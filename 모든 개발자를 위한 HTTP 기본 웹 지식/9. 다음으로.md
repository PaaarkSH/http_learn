## HTTP 에 대해서 더 깊이있게 학습

- HTTP 스팩
    - RFC 2616:  [https://tools.ietf.org/html/rfc2616](https://tools.ietf.org/html/rfc2616) → **주의! 이거 보면 안됨**
        - 인터넷에 대부분 있는 자료가 이거임
    - RFC 7230 ~ 7235: https://tools.ietf.org/html/rfc7230 → 이걸로 모두 개정
        - 2014년에 개정되고 스펙이 퍼지는데 시간이 걸리는 중이라
        - 강사님은 백엔드 개발자로써는 이 많은 양을 다 봐야되는지에 대한건 의문
- HTTP 완벽 가이드 책
    - RFC 2616 기준이라는 점 감안
    - 정석급으로 정리가 되있긴함

## 학자형 코스 vs 야생형 코스

- 학자형: 이론을 차근차근 쌓아올린 다음에 실무 활용으로 넘어가는 방식
- 야생형: 실무에서 어떤 기술을 어떤식으로 활용하는지, 깊이가 부족해도 일단 경험해본 다음에 필요해 의해서 이론 공부를 하는 방식
- 강사님은 야생형 코스를 선호하심
- 예)
    - 갑자기 실무에 들어가면 기술능력 자체가 매우 올라가고 필요해 의해서 필요한 부분을 익혀감
- 이런 관점을 볼때 JPA 강의도 실무에서 강의를 다 넣은거기 때문에 경험해 보는걸 더 추천하심
